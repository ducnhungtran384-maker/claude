## 🎯 项目概述

这是一个使用 Claude Code 和 Claude Opus 4.6 完成的项目集合，展示了 AI 在职业规划咨询和课程内容制作中的应用。

**核心案例**：中山大学临床医学大一学生转专业（软件工程）规划
- 📊 10维度筹码评估
- 🎯 多路径规划方案
- ⚠️ 风险评估与应对
- 📅 四年完整时间轴

## 💡 核心方法论

### 1️⃣ 筹码评估（10维度）
| 维度 | 说明 |
|------|------|
| 经济资本 | 家庭经济支持能力 |
| 学历信号 | 学校品牌和专业含金量 |
| 硬技能 | 可量化的技术能力 |
| 作品集 | 项目经历和产出 |
| 强关系 | 核心人脉资源 |
| 弱关系 | 跨圈连接能力 |
| 跨圈位置 | 交叉领域优势 |
| 个人标签 | 差异化竞争力 |
| 时间资本 | 可用时间和试错空间 |
| 自驱力 | 主动学习和行动力 |

### 2️⃣ 多路径规划
- **主路径**（90%精力）：最优方案，成功概率60-70%
- **备选路径**（10%精力）：Plan B/C/D，确保综合成功率90%+
- **期权思维**：即使主路径失败，准备过程也有迁移价值

### 3️⃣ 风险管理
- 识别主要风险点
- 评估发生概率
- 制定应对预案
- 反脆弱性测试

### 4️⃣ 时间轴规划
```
大一 → 大二 → 大三 → 大四
 ↓      ↓      ↓      ↓
基础   实践   深化   收获
```

## 📖 案例详解

### 学生画像
- 🏫 **学校**：中山大学（985顶尖）
- 📚 **专业**：临床医学大一
- 📊 **GPA**：中等水平
- 🌐 **英语**：四级通过
- 💪 **核心优势**：超强自驱力（能在国内用上 Claude Code + Opus 4.6）

### 核心焦虑
- 医疗环境差，学历贬值严重
- 读书时间长（本硕博8年+规培3年）
- 工作强度大，收入中等
- 不喜欢死记硬背的学习方式

### 规划方案

#### 🎯 主路径（90%精力）
**4月转专业到软件工程**
- 成功概率：60-70%
- 目标方向：软件工程 + AI + 网络安全
- 准备重点：
  - ✅ Python基础 + LeetCode刷题
  - ✅ 转专业面试话术准备
  - ✅ 展示自学能力（Claude Code使用经历）

#### 🔄 备选路径（10%精力）
1. **Plan B1**：辅修计算机/软件工程
2. **Plan B2**：本科医学 + 跨考计算机研究生
3. **Plan B3**：自学 + 实习驱动

#### 📊 概率分析
| 路径 | 成功概率 | 说明 |
|------|---------|------|
| 主路径 | 60-70% | 4月转专业 |
| 综合路径 | 90%+ | 包含所有备选方案 |

### 四年时间轴

```
📅 大一下（现在）
   ├─ 🔴 4月：转专业申请 ← 最高优先级
   └─ 🟢 暑假：Python基础 + 第一个小项目

📅 大二
   ├─ 🟡 上学期：数据结构 + 算法（面试核心）
   ├─ 🟡 下学期：参加竞赛（1-2个）
   └─ 🟡 暑假：第1段实习

📅 大三
   ├─ 🟠 上学期：确定主攻方向（AI or 安全）
   ├─ 🟠 下学期：第2段实习
   └─ 🟠 暑假：第3段实习（目标大厂）

📅 大四
   └─ 🔴 秋招/考研
```

### 核心建议

> **如果只记住一句话：**
>
> "大一发现方向不对，立刻调整，这不是失败，这是理性决策。你的目标是30岁时的位置，不是20岁时的面子。"

**精力分配**：
- 当前90%精力放在转专业准备上
- 10%精力了解Plan B
- 转专业结果出来后再调整

**最大风险**：
不是转专业失败（因为有Plan B），而是犹豫不决、拖延行动。

**核心竞争力**：
不是某个专业，而是自驱力和技术敏感度。
