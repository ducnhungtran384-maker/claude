> **学生**：中山大学 · 临床医学 · 大一  
> **核心诉求**：转专业到软件工程，走纯技术路线  
> **生成时间**：2026年4月  
> **关键时间节点**：4月转专业申请

## 一、学生画像

### 基本信息

| 维度 | 情况 |
|------|------|
| 学校/专业/年级 | 中山大学 · 临床医学 · 大一 |
| GPA | 中等水平 |
| 英语 | 四级通过 |
| 专业选择 | 自己选择，但现在后悔 |
| 已有资源 | 认识医学相关学长学姐 |
| 家庭态度 | 完全尊重选择 |
| 经济条件 | 生活费无压力，大额支出有压力 |

### 核心焦虑

- 中国医疗环境差，学历贬值严重
- 读书时间长、工作累、收入中等，投入产出比低
- 不喜欢医学的死记硬背学习方式

### 明确目标

- 转专业到软件工程
- 主攻方向：**软件工程 + AI + 网络空间安全**
- 对AI和快速迭代的技术领域有真实兴趣
- 已经在使用Claude Code和Claude Opus 4.6（展现出极强的技术探索能力和行动力）

### 关键判断

学生明确不想走"医学+AI交叉"路线。理由很清晰：大一临床医学主要学基础课（生物、化学、解剖），并未学到真正的临床知识，不足以构成跨学科优势。兴趣在于技术本身，而非"用技术解决医学问题"。这是一个非常理性的判断。

#### 转专业面试叙事

> "我高考时选择临床医学，是因为当时对医学有憧憬。但进入大学后，我发现自己真正感兴趣的是计算机技术本身。

**叙事要点**：

- ✅ 诚实表达：发现真实兴趣所在
- ✅ 积极定位：不是逃离医学，而是追寻技术热情
- ✅ 有行动证明：已经在自学，使用Claude Code
- ❌ 不要说：医学太累、赚钱少、医疗环境差

#### 面试准备重点

| 准备项 | 内容 |

#### 期权价值

即使转专业失败，准备过程也不会白费：

- Python基础 → 后续自学编程的基础
- 面试经验 → 下次申请（有些学校允许大二再转一次）
- 对自身兴趣的确认 → 更坚定地走Plan B

## 四、目标方向分析：软件工程 + AI + 网络安全

### 三者关系

```
软件工程 ─── 地基（写代码、做项目、工程能力）
    │
    ├── AI ─── 当前风口（大模型、机器学习、高薪）
    │
    └── 网络安全 ─── 长期护城河（技术门槛高、人才缺口大、AI难替代）
```

### 为什么这个组合有价值

| 方向 | 价值 | 薪资参考 |
|------|------|----------|
| 软件工程 | 基础能力，所有技术岗的根基 | 应届15-25万，3-5年30-60万 |
| AI方向 | 当前最热赛道，大模型应用需求爆发 | 应届20-40万，算法岗更高 |
| 网络安全 | 国家战略需求，人才缺口大，AI难以替代 | 应届15-25万，安全专家50-100万+ |

### 网络安全的独特优势

1. **人才缺口大**：国家战略需求，企业持续招人
2. **薪资天花板高**：顶级安全专家年薪50-100万+
3. **AI难以替代**：安全需要对抗思维和创造力
4. **入门门槛适中**：可以从CTF比赛开始，边玩边学
5. **与软件工程/AI协同**：懂开发的安全工程师更值钱

### 精力分配建议

- **软件工程**：60%精力（这是地基，必须扎实）
- **AI方向**：25%精力（紧跟前沿，做AI应用）
- **网络安全**：15%精力（作为差异化竞争力）
- **注意**：不要三个方向平均用力，会什么都不精。软件工程是基础必须扎实，AI和安全选一个作为主攻方向，另一个作为辅助

## 六、风险评估与应对

### 主要风险

| 风险 | 概率 | 应对方案 |
|------|------|----------|
| 转专业失败 | 30-40% | 启动Plan B（辅修/跨考/自学+实习） |
| AI行业泡沫破裂 | 15% | 网络安全作为对冲（反脆弱设计） |
| 就业竞争加剧 | 40% | 实习经历是最硬的护城河，大二开始积累 |
| 学习压力过大（转专业后补课多） | 50% | 优先核心课程，非核心课程过线即可 |

### 反脆弱测试

| 冲击场景 | 软件工程+AI | 软件工程+安全 |
|----------|------------|--------------|
| 经济危机/裁员潮 | 受冲击较大（AI岗可能缩招） | 受冲击较小（安全是刚需） |
| AI技术突破（替代程序员） | 需要转向AI应用层或管理层 | 反而更值钱（安全需求增加） |
| 行业政策变化 | 中性 | 利好（国家重视网络安全） |

**结论**：软件工程+AI是高收益路线，网络安全是保险路线。两者搭配构成杠铃策略。

1. **你的核心优势**：中山大学的学历信号（最大的牌） + 超强的自驱力（能用上Claude Code） + 时间充足（大一） + 方向清晰。
2. **4月转专业是第一优先级**。成功了按部就班；失败了启动辅修/自学跨考的Plan B。
3. **不要纠结"医学背景浪费了"**。大一试错成本很低，及时止损比硬撑强，你的学历信号不会贬值。
