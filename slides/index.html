🧠 核心算法：ResNet18

Core Algorithm: Deep Residual Learning

为什么选择 ResNet18？

  • ⚡
    残差连接解决梯度消失

    Skip Connection 让梯度直接回传，支持深层网络训练

  • 📦
    轻量级高效

    仅 11M 参数量，推理速度是 VGG16 的 5 倍以上

  • 🎯
    迁移学习友好

    ImageNet 预训练权重，小数据集也能快速收敛

  • 🔧
    移动端部署友好

    模型体积小，适合边缘设备与移动应用

关键技术：Transfer Learning (ImageNet) + Dynamic Data Augmentation + Weighted CrossEntropyLoss

📊 数据工程全流程

Data Engineering Pipeline

数据质量决定模型上限。本项目构建了完整的数据工程流水线，从原始数据采集到模型输入，实现全自动化处理。

📥

01. 数据采集

整合 Kaggle、OpenI 等公开数据集，收集正常与白内障眼底图像，建立原始数据仓库

🏷️

02. 标签清洗

使用 classify.py 自动化脚本，基于 Regex 规则清洗标签，修正数据注入问题

🔍

03. 去重处理

remove_duplicate.py 基于 Hashlib 计算图像 MD5，二进制级别去重，确保数据纯净

🎨

04. 动态增强

RandomRotation、ColorJitter、RandomFlip 等数据增强策略，提升模型泛化能力

Model Performance Comparison

经过多次实验对比，ResNet18 在准确率、速度、模型大小之间取得最佳平衡，被选为最终部署模型。

🎯 为什么选择 ResNet18？

虽然 DenseNet121 准确率略高 0.51%，但 ResNet18 的召回率高达 98.50%，意味着漏诊率极低，对医疗场景更为关键。同时推理速度是 VGG16 的 5 倍。

⚡ 推理性能

在 NVIDIA GTX 1660 上，单张图像推理时间 < 10ms，完全满足实时筛查需求。模型大小仅 42MB，易于移动端部署。

🔍 可解释性分析 (XAI)

Explainable AI: Grad-CAM Heatmaps

Grad-CAM 技术原理

梯度加权类激活映射 (Gradient-weighted Class Activation Mapping) 通过计算特征图的梯度，直观展示模型做出决策时关注的图像区域。

  • 🎯
    病灶定位

    热力图高亮显示晶状体混浊区域，与医生诊断区域高度一致

  • 🛡️
    增强信任

    打破"黑盒"问题，让医生理解 AI 决策依据，提升临床接受度

  • 🔬
    误差分析

    通过分析误判案例的热力图，发现模型关注偏差，指导模型优化

Grad-CAM 可视化效果

🔥
